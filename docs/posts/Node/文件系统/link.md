# link

## 硬链接（Hard Link）

硬链接的作用和用途如下：

1. **文件共享**：多个文件名指向同一个文件内容，节省空间，修改一次，所有引用都会同步变化。

2. **文件备份**：通过硬链接生成“备份名”，不复制数据，占用空间小，内容始终和原文件保持一致。

3. **文件改名（类似效果）**：创建一个新的文件名指向同一内容，不需要复制文件数据。

```js
a.txt →data
b.txt →
```

两个名字指向同一个文件数据。删除 `a.txt`后，`b.txt` 仍然可用。只有所有名字都删掉，数据才真正删除。





## 软链接（Symbolic Link / Symlink）

![image-20260224175759960](https://markdown-lai.oss-cn-hangzhou.aliyuncs.com/typora/image-20260224175759960.png)

1. **创建引用**：软链接相当于给文件或目录做一个“快捷方式”，不用复制或移动就能在别处访问它。

2. **简化路径**：可以用更短、更易记的路径，指向复杂或很深的目录结构。

3. **应对路径变化**：当目标位置变动时，只需修改软链接指向的路径，不用改动其他引用代码。

```js
b.txt → a.txt → data
```

删除 `a.txt`：b.txt 失效
