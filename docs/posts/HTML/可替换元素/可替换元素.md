# 可替换元素

内容不是由 HTML 本身决定，而是“外部资源”替换进入这个元素中，由浏览器自行决定内部内容如何渲染。

例如：

- `<img>` —— 内容来自外部图片文件
- `<video>` —— 内容来自视频资源
- `<audio>` —— 内容来自音频资源
- `<canvas>` —— 内容是 JS 绘制的画布
- `<iframe>` —— 内容来自另一个 HTML 页面
- `<input>` —— 内容来自浏览器实现的控件
- `<embed>` / `<object>` —— 插件/资源内容

这些都不是由 HTML 标签结构渲染出的内容，而是浏览器“替换”进来的，所以叫 **可替换元素**。

| 特性                  | 可替换元素                     | 普通元素（非替换）              |
| --------------------- | ------------------------------ | ------------------------------- |
| 内容来源              | 外部资源（图片/视频/控件）     | HTML 内容                       |
| 宽高行为              | 有固有宽高（例如图片原始像素） | 默认为自动撑开内容              |
| 能否用 `object-fit`？ | ✔ 可以（作用于内部替换内容）   | ❌ 不行                          |
| baseline 行为         | 默认以内容底部对齐             | 通常根据文字基线                |
| 可完全被浏览器控制    | ✔                              | ❌                               |
| 用户代理样式强        | ✔                              | 较弱，例如 input 样式难完全重置 |

可替换元素通常有：

- **固有宽度（Intrinsic width）**
- **固有高度（Intrinsic height）**
- **固有比例（Intrinsic ratio）**

`<img>` 的固有尺寸来自图片文件的分辨率

如果设置：

```
<img src="image.jpg">
```

浏览器会用图片原始大小渲染。

这就是为什么 `<img>` 即使是 inline，也能设置：

```css
width: 200px;
height: 200px;
```

## 内容盒

当你给 `<img>` 设置 `width` 和 `height`，就形成了一个固定大小的 **内容盒**（CSS box 的 content area）。

这个内容盒就是 `object-fit` 的参考容器。

`object-fit: fill` 就是让图片拉伸或压缩，填满这个 **内容盒**，不管原始比例。
