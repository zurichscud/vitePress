# 类加载器

## 类加载器

| **加载器类型**              | **加载路径 / 职责**                 | **getClassLoader() 的结果**                            |
| --------------------------- | ----------------------------------- | ------------------------------------------------------ |
| **Bootstrap ClassLoader**   | 加载核心类库（如 `java.lang.*`）    | **null** (原生代码实现)                                |
| **Extension ClassLoader**   | 加载扩展目录 `lib/ext` 中的类       | `jdk.internal.loader.ClassLoaders$PlatformClassLoader` |
| **Application ClassLoader** | 加载用户项目路径（ClassPath）下的类 | `jdk.internal.loader.ClassLoaders$AppClassLoader`      |

## 获取类加载器

### 应用（系统）类加载器

- 方式一：通过实例对象

```java
ClassLoader loader1 = obj.getClass().getClassLoader();//sun.misc.Launcher$AppClassLoader@18b4aac2
```

- 方式二：通过自定义类的名字面量

```java
ClassLoader loader2 = MyService.class.getClassLoader();//sun.misc.Launcher$AppClassLoader@18b4aac2
```

- 方式三

```java
ClassLoader threadLoader = Thread.currentThread().getContextClassLoader();//sun.misc.Launcher$AppClassLoader@18b4aac2
```

- 方式四

```java
ClassLoader sysLoader = ClassLoader.getSystemClassLoader();//sun.misc.Launcher$AppClassLoader@18b4aac2
```

### 获取父Classloader

- 通过系统核心类获取

Object是Java核心库中的类，由Bootstrap ClassLoader加载

```java
Object o = new Object();
ClassLoader classLoader1 = o.getClass().getClassLoader();
```

- 通过用户自定义类获取

```java
public class LoaderDemo {
    public static void main(String[] args) {
        // 1. AppClassLoader (System ClassLoader)
        ClassLoader loader = LoaderDemo.class.getClassLoader();
        System.out.println("当前类加载器: " + loader);

        // 2. ExtClassLoader (Platform ClassLoader in Java 9+)
        System.out.println("ExtClassLoader: " + loader.getParent());

        // 3. Bootstrap ClassLoader
        // 注意：由于它由 C++ 编写，在 Java 中返回 null
        System.out.println("BootstrapClassLoader: " + loader.getParent().getParent());
    }
}
```



## 加载类

```java
loader.loadClass("com.xxx.User");
Class.forName("com.xxx.User");
```

- **`loadClass`**：默认不会执行类的初始化（即不会运行 `static` 静态代码块）。

- **`Class.forName("...")`**：默认会执行类的初始化，常用于数据库驱动加载（触发 `DriverManager.registerDriver`）



## 双亲委派机制

**双亲委派机制（Parents Delegation Model）** 是 Java 类加载器（ClassLoader）在加载类时遵循的一种核心设计模式。

**一个类只会被一个Classloader加载，具体是哪个Classloader，取决于双亲委派机制**。

当一个类加载器收到加载类的请求时，它首先不会自己尝试去加载，而是将这个请求“委派”给它的父类加载器去完成。每一层都是如此，因此所有的加载请求最终都应该传送到顶层的启动类加载器（Bootstrap ClassLoader）中。只有当父类加载器反馈自己无法完成这个加载请求（它的搜索范围中没有找到所需的类）时，子加载器才会尝试自己去加载。

::: tip 双亲委派机制的必要性

如果没有双亲委派，用户可以随意编写一个名为 `java.lang.Object` 的类并存放在 ClassPath 下。由于 `Application ClassLoader` 会直接加载它，导致系统中原本稳定的 `Object` 类被替换。有了双亲委派，加载 `java.lang.Object` 的请求最终会由 `Bootstrap ClassLoader` 处理，它只会加载 Java 核心库里的那个类，从而保证了 Java 运行环境的安全。

通过委派机制，确保了同一个类在被不同的加载器加载时，最终都会由最合适的父加载器完成，保证了类在 JVM 中的全局唯一性。

:::
