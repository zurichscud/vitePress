# 组合注解

组合注解就是**在一个新定义的注解上标注一个或多个现有的注解**，从而将这些注解的功能“打包”在一起。

## 基本语法

在一个注解的定义上使用了其他注解

## 为什么需要组合注解

在 Spring 中，如果你有一个常用的配置组合，比如定义一个 Rest 控制器并处理跨域，每次都写 `@RestController` 和 `@CrossOrigin` 很麻烦。你可以组合它们：

```java
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@RestController
@CrossOrigin(origins = "*") // 预设默认配置
public @interface MyApi {
    // 这样，只需标注 @MyApi 就能同时获得两个注解的功能
}
```



## @GetMapping

`@GetMapping` 就是嵌套了 `@RequestMapping` 注解，组合了功能

```java
// @GetMapping 的底层逻辑
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
@RequestMapping(method = RequestMethod.GET) // 明确了请求方式
public @interface GetMapping {
    String value() default "";
}
```
