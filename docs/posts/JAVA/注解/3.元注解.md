# 元注解

元注解（Meta-Annotation）就是：**用来修饰“注解”的注解**。它决定了你自定义注解的**作用范围、生命周期、行为特性**。

## @Target

限定你的注解可以出现的位置

```java
@Target(ElementType.METHOD)
```

| ElementType       | 作用位置                 |
| ----------------- | ------------------------ |
| `TYPE`            | 类、接口、枚举、注解类型 |
| `FIELD`           | 成员变量（字段）         |
| `METHOD`          | 方法                     |
| `PARAMETER`       | 方法参数                 |
| `CONSTRUCTOR`     | 构造方法                 |
| `LOCAL_VARIABLE`  | 局部变量                 |
| `ANNOTATION_TYPE` | 注解类型本身             |
| `PACKAGE`         | 包                       |
| `TYPE_PARAMETER`  | 泛型类型参数             |
| `TYPE_USE`        | 泛型/类型使用的任何地方  |

## @Retention

决定了注解的生命周期

```java
@Retention(RetentionPolicy.RUNTIME)
```

| 类型    | 说明                                   | 场景       |
| ------- | -------------------------------------- | ---------- |
| SOURCE  | 注解仅存在于源代码中，在字节码中不包含 | Lombok     |
| CLASS   | 在字节码中存在，但是运行时无法获取     | 默认       |
| RUNTIME | 运行时可获取，可以通过反射获取         | Spring/AOP |



## @Documented

注解是否出现在 Javadoc 文档中

```java
@Documented
public @interface Log {}
```



## @Inherited

子类是否继承父类的注解

```java
@Inherited
@Target(ElementType.TYPE)
public @interface MyAnno {}
```

```java
@MyAnno
class Parent {}

class Child extends Parent {}
```

```java
Child.class.isAnnotationPresent(MyAnno.class) // true
```

## @Repeatable

**`@Repeatable`**，它是 Java 8 新增的，用于支持 **同一个注解在同一个位置重复使用**

### 数组属性注解缺陷

不方便单独处理，假如要给每个角色加额外属性（如权限级别、描述、策略等），数组就不够了

```java
public @interface Role {
    String value();
    int level() default 1;
}

@Role({"ADMIN", "USER"}) // ❌ 不能同时给每个角色不同 level
```

### 使用嵌套注解

```java
// 容器注解
public @interface Roles {
    Role[] value();
}

// 具体注解
public @interface Role {
    String value();
    Integer level();
}

// 使用时
@Roles({@Role("ADMIN",level=5), @Role("USER",level=1)})
public void test() {}
```



### 定义可重复注解

1. 定义具体注解

```java
import java.lang.annotation.Repeatable;

@Repeatable(Roles.class) // 指定容器注解
public @interface Role {
    String value();
}
```

2. 定义容器注解

```java
public @interface Roles {
    Role[] value();
}
```

3. 使用

```java
@Role(value="ADMIN", level=5)
@Role(value="USER", level=1)
```
