# 嵌套注解

注解的属性类型可以是另一个注解。**嵌套注解**是指在一个注解的定义中，将另一个注解作为其成员变量的类型。

## 基本语法

要实现嵌套注解，首先需要定义“子注解”，然后在“父注解”中像引用普通类型（如 `String` 或 `int`）一样引用它。

```java
// 1. 定义子注解
public @interface Review {
    String reviewer();
    String comment() default "Passed";
}

// 2. 在父注解中嵌套子注解
public @interface Task {
    String description();
    Review review(); // 嵌套子注解
}
```

在使用时，嵌套注解的赋值方式类似于对象的实例化，使用 `@注解名(...)` 的语法：

```java
@Task(
    description = "完成登录模块",
    review = @Review(reviewer = "张三", comment = "代码需重构")
)
public class LoginService {
    // ...
}
```

## 为什么需要嵌套注解

从功能上看，把所有成员都拍平放在一个注解里完全能跑通，但**嵌套注解**存在的意义主要在于**逻辑解耦、复用性**以及**表达复杂的层级关系**。

如果一个注解需要配置的参数非常多（比如超过 10 个），直接写在一起会让注解变得臃肿且难以维护。

- 拍平写法：

```java
@ServiceConfig(
    dbUrl = "...", dbUser = "...", dbPwd = "...", 
    cacheHost = "...", cachePort = "...", cacheExpiry = 60
)
```

- 嵌套写法：

通过嵌套，你可以一眼看出哪些参数是属于数据库的，哪些是属于缓存的。

```java
@ServiceConfig(
    db = @DbConfig(url = "...", user = "...", pwd = "..."),
    cache = @CacheConfig(host = "...", port = "...", expiry = 60)
)
```

嵌套的“子注解”可以作为独立的个体在别处复用。

## 反射获取嵌套注解

在运行时，你可以通过反射轻松获取嵌套的内容：

```java
Task task = LoginService.class.getAnnotation(Task.class);
System.out.println("描述: " + task.description());

// 获取嵌套的子注解
Review review = task.review();
System.out.println("审核人: " + review.reviewer());
```

## 与@Repeatable 对比

虽然嵌套注解（通过数组形式）可以实现“重复注解”的效果，但 `@Repeatable` 提供了更优雅的语法糖，允许你在同一个元素上多次写同一个注解，而不需要显式地写出容器注解。

- **嵌套式**：`@Filters({ @Filter(A), @Filter(B) })`

- **重复式**：`@Filter(A) @Filter(B)` （底层依然依赖嵌套容器）
