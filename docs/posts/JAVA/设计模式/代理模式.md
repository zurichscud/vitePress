# 代理模式

## 静态代理

Java 静态代理，本质就是：**在编译期就已经确定代理类的实现**，通过“代理类”去间接调用目标对象，从而在不修改原有代码的情况下增强功能。

### 实现

- **接口（Subject）** 定义目标方法

```java
public interface UserService {
    void save();
}
```



- **目标类（RealSubject）** 真正干活的类

```java
public class UserServiceImpl implements UserService {
    @Override
    public void save() {
        System.out.println("保存用户");
    }
}
```



- **代理类（Proxy）**对目标类进行“增强”（比如加日志、权限校验等）

```java
public class UserServiceProxy implements UserService {

    private UserService target;

    public UserServiceProxy(UserService target) {
        this.target = target;
    }

    @Override
    public void save() {
        System.out.println("前置增强：日志记录");

        target.save();  // 调用真实对象

        System.out.println("后置增强：事务处理");
    }
}
```

### 缺陷

- **每个接口都要写一个代理类（类爆炸）**

- 方法多了，代理类也要全部实现

- 不够灵活
