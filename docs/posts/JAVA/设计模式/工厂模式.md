# 工厂模式

## 简单工厂模式（静态工厂方法）

### 实现

- 抽象产品

```java
interface Product {
    void use();
}
```

- 具体产品

```java
class ProductA implements Product {
    public void use() {
        System.out.println("使用A产品");
    }
}

class ProductB implements Product {
    public void use() {
        System.out.println("使用B产品");
    }
}
```

- 工厂类

```java
class Factory {
    public static Product create(String type) {
        if ("A".equals(type)) {
            return new ProductA();
        } else if ("B".equals(type)) {
            return new ProductB();
        }
        throw new IllegalArgumentException("未知类型");
    }
}
```



- 使用

```java
public class Main {
    public static void main(String[] args) {
        Product p = Factory.create("A");
        p.use();
    }
}
```

### 缺陷

- 违反开闭原则（OCP）

- 如果你要新增 `ProductC`,必须修改工厂类

```java
class Factory {
    public static Product create(String type) {
        if ("A".equals(type)) {
            return new ProductA();
        } else if ("B".equals(type)) {
            return new ProductB();
        } else if ("C".equals(type)) {
            return new ProductC();
}
        throw new IllegalArgumentException("未知类型");
    }
}
```

- 工厂类一旦出现问题，整个系统将全部瘫痪



## 工厂方法模式

把“创建对象”的职责，从一个大工厂，拆分到多个子工厂中**。它是对“简单工厂”的升级版，核心目标：**符合开闭原则（OCP）**

### 实现

- 抽象工厂

```java
interface Factory {
    Product create();
}
```

- 具体工厂

```java
class ProductAFactory implements Factory {
    public Product create() {
        return new ProductA();
    }
}

class ProductBFactory implements Factory {
    public Product create() {
        return new ProductB();
    }
}
```

- 使用

```java
public class Main {
    public static void main(String[] args) {
        Factory factory = new ProductAFactory();
        Product product = factory.create();
        product.use();
    }
}
```

想换产品，只需要换工厂：

```java
Factory factory = new ProductBFactory();
```



### 缺陷

- 类数量爆炸（一个产品一个工厂）

- 结构更复杂

- 对简单场景来说有点“过度设计”
