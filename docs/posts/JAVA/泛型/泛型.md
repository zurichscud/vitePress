# 泛型

**泛型不支持基本数据类型（如 `int`、`double`）**，只能使用**引用类型（对象）**。

## 类型擦除

泛型是在编译期实现的（**类型擦除**机制）：

```java
List<Integer> list = new ArrayList<>();
```

编译后会变成：

```java
List list = new ArrayList();
```

- 泛型类型信息在运行时被“擦除”

- JVM 只认识 **Object**

- 而基本类型（`int`）不是对象 ❌

## 为什么需要泛型

### 使用泛型之前

```java
List list = new ArrayList();
list.add("hello");
list.add(123); // 也能加

String str = (String) list.get(1); // 运行时才报错
```

- 类型不安全（什么都能放）
- 需要强制类型转换
- 错误发生在运行时

### 使用泛型之后

```java
List<String> list = new ArrayList<>();
list.add("hello");
// list.add(123); // 编译直接报错

String str = list.get(0); // 不需要强转
```

- 编译期检查类型 
- 不需要强制转换 
- 代码更清晰 

## 泛型类

```java
public class Box<T> {
    private T value;

    public void set(T value) {
        this.value = value;
    }

    public T get() {
        return value;
    }
}
```

创建对象时指定泛型参数：

```java
Box<String> box = new Box<>();
box.set("hello");
String val = box.get();
```

## 泛型方法

泛型参数 `<T>` 需要写在返回值前面，表示这个方法是一个泛型方法，泛型参数来着方法调用者，而不是来自`class`

```java
public <T> void test(T a, T b) {}
```

泛型参数根据传入的形参自动推断出类型

```java
test(1, 2);       // T = Integer
test("a", "b");   // T = String
```



## 泛型接口

```java
public interface Converter<T> {
    T convert(String input);
}
```

实现接口时，指定泛型接口的参数：

```java
public class IntegerConverter implements Converter<Integer> {
    public Integer convert(String input) {
        return Integer.valueOf(input);
    }
}
```

## 泛型与静态成员

### 泛型类

静态成员不能使用类的泛型，是因为在泛型类中，泛型属于“对象”，而静态属于“类”

```java
public class Test<T> {
    static T value; // ❌ 编译报错
}
```

泛型类中的泛型是属于实例级别的：

```java
Test<String> t1 = new Test<>();
Test<Integer> t2 = new Test<>();
```

### 泛型方法

对于泛型方法，泛型属于方法的，因此可以定义一个静态泛型方法

```java
public class Test {

    public static <T> void print(T t) {
        System.out.println(t);
    }
}
```

## 泛型与多态

- `List<Animal>`和`List<Dog>`没有任何关系，不存在多态的关系

假设 Java 允许这种赋值，会出现类型安全漏洞：

```java
List<Dog> dogs = new ArrayList<>();
List<Animal> animals = dogs; // 假设这行能编译通过

animals.add(new Cat()); // 此时你往“狗狗列表”里塞进了一只猫
Dog myDog = dogs.get(0); // 这里会抛出 ClassCastException，因为拿到的是猫
```

为了防止这种“指鹿为马”的情况，Java 编译器在编译阶段就会阻止 `List<Animal> list = new ArrayList<Dog>();`。

- 当泛型一致，`List<Animal>`和`ArrayList<Animal>`存在多态关系

- `List<Animal>`允许Dog对象的添加：

```java
List<Animal> list
```

编译器会认为：

```java
boolean add(Animal e)
```

e允许Animal的子类赋值，这是多态的一种表现

## 通配符

如果你需要编写一个既能处理 `List<Animal>` 又能处理 `List<Dog>` 的方法，需要使用 **通配符**

| 类型 | 写法            | 能读   | 能写 |
| ---- | --------------- | ------ | ---- |
| 无界 | `<?>`           | Object | ❌    |
| 上界 | `<? extends T>` | T      | ❌    |
| 下界 | `<? super T>`   | Object | ✅    |

### 无界通配符 `?`

**未知类型（可以是任何类型）**：可以读取数据，不允许写入数据

```java
List<?> list =null;
List<String> list2 = new ArrayList<>();

list2.add("haha")
list =list2;
Object obj = list.get(0); // ✅：haha
list.add("abc");          // ❌
```

虽然我们肉眼看到它指向的是 `List<String>`，但编译器为了保证绝对安全，会进行如下逻辑推理：

- `list` 可能指向 `List<String>`。
- `list` 也可能指向 `List<Integer>` 或 `List<Dog>`。
- 如果我允许你往 `list` 里添加 `"abc"`（String），万一它实际指向的是 `List<Integer>`，那程序在运行时就会崩溃。

**结论：** 为了堵死这个漏洞，Java 规定不能向 `List<?>` 添加任何对象（除了 `null`）。

### 上界通配符 `? extends T`

T类型或T类型的子类：可以读取数据，不允许写入数据

```java
List<? extends Animal> list = new ArrayList<Dog>();
Animal a = list.get(0); // ✅
list.add(new Dog());    // ❌
```

- list不允许添加元素
- list不知道到底是 `List<Dog> `还是` List<Cat>`，如果Dog类型的对象进入` List<Cat>`类型不安全。

### 下界通配符 `? super T`

T类型或T类型的父类：可以读取数据，允许写入T类型或T类型的子类

```java
List<? super Dog> list = new ArrayList<Animal>();
list.add(new Dog());   // ✅
list.add(new Teddy()); // ✅

Object obj = list.get(0); // ✅
```

- list允许添加元素
- 不管运行时类是`ArrayList<Animal>`还是`ArrayList<Dog>`都不会出现类型安全问题
