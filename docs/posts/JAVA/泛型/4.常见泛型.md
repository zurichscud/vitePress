# 常见泛型

## Function<? super T, ? extends K>

接收 T 或 T 的父类，返回 K 或 K 的子类

- 只要这个函数能处理 T 就行，所以它的参数可以更“宽泛”（父类也能接）

- 返回的东西必须是 K 或更具体的类型（子类）。因为调用方要“拿来用”这个返回值，必须保证它 **至少是 K**（或者更具体的子类）

```java
class Animal {}
class Dog extends Animal {}
class Teddy extends Dog {}
```

- 形参

```java
Function<? super Dog, String> func;
```

对于形参：我们希望能够有更多的参数传入。

```java
Function<Dog, String> f1 = d -> "dog";
Function<Animal, String> f2 = a -> "animal";
Function<Object, String> f3 = o -> "object";
```

- 返回值

```java
Function<Dog, ? super Teddy> func;
```

返回值是不知道是Object还是Animal，还是Teddy，调用者只能使用Object的通用方法才能保证安全，这是不合理的。因此只能使用`? extends K`。即返回值需要约束上限
