# 泛型的不变性



## 泛型不变性

- `List<Animal>`和`List<Dog>`没有任何关系，不存在多态的关系

假设 Java 允许这种赋值，会出现类型安全漏洞：

```java
List<Dog> dogs = new ArrayList<>();
List<Animal> animals = dogs; // 假设这行能编译通过

animals.add(new Cat()); // 此时你往“狗狗列表”里塞进了一只猫
Dog myDog = dogs.get(0); // 这里会抛出 ClassCastException，因为拿到的是猫
```

为了防止这种“指鹿为马”的情况，Java 编译器在编译阶段就会阻止 `List<Animal> list = new ArrayList<Dog>();`。

当泛型一致，`List<Animal>`和`ArrayList<Animal>`存在多态关系

## 通配符

如果你需要编写一个既能处理 `List<Animal>` 又能处理 `List<Dog>` 的方法，需要使用 **通配符**

![image-20260330105152816](https://markdown-lai.oss-cn-hangzhou.aliyuncs.com/typora/image-20260330105152816.png)



- 类型参数（形参）

类似方法/类的“**占位符**”，调用时由外部具体指定。

就像方法的形参：你可以给它一个名字 `T`，在类/方法里反复使用。

```java
class Cage<T extends Animal> {
    private T occupant;
    public void setOccupant(T t) { this.occupant = t; }
    public T getOccupant() { return occupant; }
}
Cage<Dog> dogCage = new Cage<>(); // Dog 就是 T 的具体实参
```

- 通配符

通常出现在 **变量、方法参数、返回值**，表示“某种 Animal 的子类型，但不关心具体是什么”。

可以理解成“**我不指定类型，只知道它是 Animal 或子类**”，相当于“实参的类型未知，但符合约束”。

```java
List<? extends Animal> list = new ArrayList<Dog>();
```



## 无界通配符 `?`

**未知类型（可以是任何类型）**：可以读取数据，不允许写入数据

```java
List<?> list =null;
List<String> list2 = new ArrayList<>();

list2.add("haha")
list =list2;
Object obj = list.get(0); // ✅：haha
list.add("abc");          // ❌
```

虽然我们肉眼看到它指向的是 `List<String>`，但编译器为了保证绝对安全，会进行如下逻辑推理：

- `list` 可能指向 `List<String>`。
- `list` 也可能指向 `List<Integer>` 或 `List<Dog>`。
- 如果我允许你往 `list` 里添加 `"abc"`（String），万一它实际指向的是 `List<Integer>`，那程序在运行时就会崩溃。

**结论：** 为了堵死这个漏洞，Java 规定不能向 `List<?>` 添加任何对象（除了 `null`）。

## 上界通配符 `? extends T`（协变）

T类型或T类型的子类：可以读取数据，不允许写入数据

```java
List<? extends Animal> list = new ArrayList<Dog>();
Animal a = list.get(0); // ✅
list.add(new Dog());    // ❌
```

- list不允许添加元素
- list不知道到底是 `List<Dog> `还是` List<Cat>`，如果Dog类型的对象进入` List<Cat>`类型不安全。

## 下界通配符 `? super T`（逆变）

T类型或T类型的父类：可以读取数据，允许写入T类型或T类型的子类

```java
List<? super Dog> list = new ArrayList<Animal>();
list.add(new Dog());   // ✅
list.add(new Teddy()); // ✅

Object obj = list.get(0); // ✅
```

- list允许添加元素
- 不管运行时类是`ArrayList<Animal>`还是`ArrayList<Dog>`都不会出现类型安全问题

## PECS

**P**roducer **E**xtends (生产者使用 `extends`)： 如果你只需要从集合中**读取**数据（集合是数据的生产者），使用 `? extends T`。

**C**onsumer **S**uper (消费者使用 `super`)： 如果你只需要往集合中**写入**数据（集合是数据的消费者），使用 `? super T`。

| 类型 | 写法            | 能读   | 能写 |
| ---- | --------------- | ------ | ---- |
| 无界 | `<?>`           | Object | ❌    |
| 上界 | `<? extends T>` | T      | ❌    |
| 下界 | `<? super T>`   | Object | ✅    |

```java
public static <T> void copy(List<? super T> dest, List<? extends T> src) {
    for (T item : src) { // src是生产者，从中读取T（或子类）对象
        dest.add(item);   // dest是消费者，向其中写入T（或父类）对象
    }
}
```
