# 基本原理

 Quartz 的核心只需要掌握三个基本概念：**Job（干什么）**、Trigger（什么时候干）和 **Scheduler（谁来协调）**

## install

```xml
<dependency>
    <groupId>org.quartz-scheduler</groupId>
    <artifactId>quartz</artifactId>
    <version>2.3.2</version>
</dependency>
```



## 核心架构

Quartz 的运作流程可以简单理解为：你定义一个具体的任务，给它配上一把“闹钟”，然后交给一个管家去管理。

- **Job**: 接口，实际要执行的业务逻辑。
- **JobDetail**: 用于定义 Job 的实例属性（如名称、组名）。
- **Trigger**：触发器，定义了任务的运行规则。它决定了任务在什么时间启动、停止以及执行的频率。
- **Scheduler**: 调度中心，将 JobDetail 和 Trigger 绑定在一起。

![image-20260511142253041](https://markdown-lai.oss-cn-hangzhou.aliyuncs.com/typora/image-20260511142253041.png)

- **Job** 与 **Trigger** 的关系是1：N的关系

  例如，你可以定义一个“清理日志”的 Job，然后给它配两个触发器：一个每小时执行一次，另一个在特定的节假日凌晨执行。

  





## QuickStart

实现 `org.quartz.Job` 接口：

```java
public class MyJob implements Job {
    @Override
    public void execute(JobExecutionContext context) {
        System.out.println("任务执行了，当前时间: " + new java.util.Date());
    }
}
```



```java
public class QuartzTest {
    public static void main(String[] args) throws SchedulerException {
        // 1. 创建 JobDetail 实例，绑定 Job 实现类
        JobDetail job = JobBuilder.newJob(MyJob.class)
                .withIdentity("myJob", "group1")
                .build();

        // 2. 创建 Trigger，设置每 5 秒执行一次
        Trigger trigger = TriggerBuilder.newTrigger()
                .withIdentity("myTrigger", "group1")
                .startNow()
                .withSchedule(SimpleScheduleBuilder.simpleSchedule()
                        .withIntervalInSeconds(5)
                        .repeatForever())
                .build();

        // 3. 获取调度器并启动
        Scheduler scheduler = StdSchedulerFactory.getDefaultScheduler();
        scheduler.start();
        
        // 4. 将任务和触发器注册到调度器
        scheduler.scheduleJob(job, trigger);
    }
}
```



## 持久化配置

Quartz 的数据库表**不是自动生成的**（不像 JPA/Hibernate 可以自动建表），而是通过 Quartz 官方提供的 **SQL 脚本** 来创建的

- **表前缀**：默认为 `QRTZ_`，可在 `quartz.properties` 修改。

- **是否自动创建**：Quartz 本身不会自动建表，你必须先运行脚本。
