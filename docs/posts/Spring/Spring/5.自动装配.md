# 自动装配

Spring 的**自动装配（Autowiring）**是 Spring 框架核心特性 IOC（控制反转）的一种具体实现方式。它允许 Spring 容器在创建 Bean 时，自动将相互依赖的组件注入到一起，而不需要开发者手动在 XML 或 Java 配置中进行显式引用。

自动装配是基于set注入的。

## byType

按类型装配，这是 Spring **默认且最常用**的逻辑（例如 `@Autowired` 的默认行为）。

Spring 容器寻找与属性类型 **完全一致** 或 **兼容（子类/实现类）** 的 Bean。

```xml
<bean id="dao1" class="com.test.UserDao"/>

<bean id="userService"
      class="com.test.UserService"
      autowire="byType"/>
```

如果容器中存在**多个**相同类型的 Bean，Spring 会直接报错 `NoUniqueBeanDefinitionException`，因为它不知道该选哪一个。





## byName

按属性名装配，Spring 会查看属性的 **变量名**，并去容器中寻找 ID（id就是bean的名称） 与之完全一致的 Bean。

```java
public class UserService {
  //寻找名为userDao的bean
    private UserDao userDao;

    public void setUserDao(UserDao userDao) {
        this.userDao = userDao;
    }
}
```



```xml
<bean id="userDao" class="com.test.UserDao"/>

<bean id="userService" class="com.test.UserService" autowire="byName"/>
```

## 对比

| **特性**       | **byType**                       | **byName**                      |
| -------------- | -------------------------------- | ------------------------------- |
| **匹配标准**   | 类或接口的类型 (Class/Interface) | Bean 的唯一标识符 (ID/Name)     |
| **唯一性要求** | 容器中该类型只能有一个 Bean      | 容器中该 ID 必须唯一            |
| **默认注解**   | `@Autowired`                     | `@Resource`                     |
| **冲突处理**   | 报错（如果存在多个同类 Bean）    | 报错（如果没有找到对应 ID）     |
| **耦合度**     | 较低（不依赖具体名称）           | 稍高（变量名需与 Bean ID 对应） |
