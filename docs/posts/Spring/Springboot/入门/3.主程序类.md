# 主程序类

在 **Spring Boot** 中，主程序类（启动类）默认会作为组件扫描的起点，**自动扫描该类所在包及其所有子包中的组件**。

这个行为主要由 **`@SpringBootApplication`** 控制。

## @SpringBootApplication

`@SpringBootApplication` 是一个组合注解，内部包含三个核心注解：

```java
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@Documented
@Inherited
@SpringBootConfiguration
@EnableAutoConfiguration
@ComponentScan(excludeFilters = { @Filter(type = FilterType.CUSTOM, classes = TypeExcludeFilter.class),
		@Filter(type = FilterType.CUSTOM, classes = AutoConfigurationExcludeFilter.class) })
public @interface SpringBootApplication {}
```

### @ComponentScan

指定了Springboot应用的包扫描路径

| **组件**                             | **排除对象**                      | **核心目的**                                       |
| ------------------------------------ | --------------------------------- | -------------------------------------------------- |
| **`TypeExcludeFilter`**              | 测试类或用户自定义排除项          | 提供灵活性，支持切面测试和动态排除。               |
| **`AutoConfigurationExcludeFilter`** | 自动配置类 (`@AutoConfiguration`) | 防止自动配置与普通组件扫描冲突，维护配置加载顺序。 |

## @EnableAutoConfiguration

`@EnableAutoConfiguration`是Springboot核心注解，提供自动引入配置的功能

### 核心逻辑

**“既然你引入了这个 Jar 包，我就猜你大概率要用它，所以我帮你把配置写好。”**

- 如果你引入了 `spring-boot-starter-web`，它会自动配置 Tomcat 和 Spring MVC。
- 如果你引入了 `spring-boot-starter-data-redis`，它会自动配置 `RedisTemplate`。

### 原理

当你标注了 `@EnableAutoConfiguration`（或者直接用 `@SpringBootApplication`）时，Spring 会触发一个关键组件：`AutoConfigurationImportSelector`。

它的工作流程如下：

1. **寻找候选名单**：它会去扫描所有 Jar 包下的 `META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports` 文件（老版本是 `spring.factories`）。
2. **加载配置类**：这些文件里记录了几百个全限定类名，比如 `RabbitAutoConfiguration`、`DataSourceAutoConfiguration` 等。
3. **条件过滤**：这是最关键的一步。它不会把这几百个配置全部加载，而是根据 `@ConditionalOnClass` 等注解判断：**“你的类路径下有 MySQL 驱动吗？有的话我才加载 DataSource 配置。”**

### 为什么需要排除

因为自动配置类（标注了 `@AutoConfiguration` 的类）本质上也是 Spring 的配置类。如果不加排除过滤器：

1. `@ComponentScan` 可能会扫描到这些类。
2. 这些类会被当做普通的 `@Component` 提前加载。
3. 这会绕过自动配置特有的**顺序控制**和**条件判断**，导致你的自定义配置和自动配置打架。

## @SpringBootConfiguration

```java
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@Documented
@Configuration
@Indexed
public @interface SpringBootConfiguration {}
```

因此被 `@SpringBootApplication`标注的类是一个配置类
