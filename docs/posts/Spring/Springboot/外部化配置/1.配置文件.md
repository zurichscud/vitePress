# 配置文件

配置文件中的值，最终会绑定在某个类上。这个类会在容器中创建对象

## 外部化配置

外部化配置是指，将配置信息存储在应用之外，这样配置信息就可以独立于代码进行管理。这样方便了配置的修改，并且修改hiya不需要重新编译代码，也不需要重新部署项目。

Springboot支持多种外部化配置方式

## application.properties

`application.properties`配置文件是Springboot默认的配置文件。`application.properties`不是必须的，因为自动配置提供了一系列的默认配置。没有`application.properties`也可以正常运行Springboot程序。如果需要更改这些默认行为，就需要在`application.properties`配置了。



## 默认约定（自动识别）

Spring Boot 默认只会自动加载：

```
application-{profile}.yml
application-{profile}.properties
```

不会自动加载：

```
my-dev.yml ❌
```



## 配置文件优先级

SpringBoot在启动时，会尝试从如下位置加载配置文件：

1. `./config/`：SpringBoot会在当前工作的config文件夹中寻找
2. `./`：如果在`./config`中找不到，则会从当前工作目录中寻找
3. `classpath:/config/`：如果在上述目录中没有找到，则会在类路径下的config目录中寻找
4. `classpath:/`：如果上述目录中没有找到，则会在类路径下寻找

Springboot会根据上述顺序加载所有配置文件，当**多个位置存在相同配置项**时，高优先级的覆盖低优先级的，类似于CSS。

```java
/*优先级从高到低*/
./config/application.yml
./application.yml
classpath:/config/application.yml
classpath:/application.yml
```

SpringBoot会先找位置，再找类型。对于同位置的配置文件，`properties`比`yaml`的优先级更高

我们可以显式指定配置文件，只加载你指定的这个配置文件：

- 指定配置文件名

```sh
java -jar app.jar --spring.config.name=myapp
```

- 指定配置文件路径

```sh
java -jar app.jar --spring.config.location=/opt/config/application.yml
```



## 配置拆分

后导入的覆盖先导入的，当前文件优先级最高

```yaml
spring:
  config:
    import:
      - classpath:common.yml
      - classpath:db.yml
```

- 先加载 common.yml
- 再加载 db.yml
- 最后再加载当前 application.yml

## 多环境切换



### 基础用法

- 定义多环境配置文件

```
application.yml           # 公共配置
application-dev.yml       # 开发环境
application-test.yml      # 测试环境
application-preprod.yml      # 预生产环境
application-prod.yml      # 生产环境
```

-  在主配置中指定环境

```yaml
spring:
  profiles:
    active: dev
```

- 实际生效规则

```
application.yml
+
application-dev.yml（覆盖）
```

### 命令行

```sh
java -jar app.jar --spring.profiles.active=prod
```



## Environment

`Environment` 属于 Spring， **提供配置读取能力 + 环境信息判断能力**。

`Environment` 读取的是 **最终生效的值（经过优先级覆盖后的结果）**，不是某一个配置文件里的原始值。

- 可以读取所有SpringBoot配置值

```java
@Autowired
private Environment environment;

public void test() {
    String port = environment.getProperty("server.port");
    System.out.println(port);
}
```

- 环境隔离

```java
if (environment.acceptsProfiles("dev")) {
    System.out.println("当前是开发环境");
}
```
