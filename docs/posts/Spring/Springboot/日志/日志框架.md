# 日志框架

## 为什么需要日志框架

频繁使用`System.out.println()`会占用额外的CPU资源，尤其是高并发的场景下，可能会影响应用的性能

控制台输出无法长期保存和检索

缺乏灵活性



## SLF4J

SLF4J （Simple Logging Facade for Java）是一个日志接口。常见的实现：

### Log4j

Apache维护的古老日志项目

### Log4j2

Log4j2是Log4j的下一代版本，性能更优

### Logback

Log4j的创始人开发的另一个日志框架。比Log4j的性能更好

### JUL

Java SE中的日志框架，不需要额外的库文件



## Springboot日志

### 默认日志方案

- Spring Boot 默认日志体系：SLF4J（接口） + Logback（实现）
- 默认日志级别是INFO

### 日志框架切换

```xml
<!-- 排除默认日志 -->
<exclusions>
    <exclusion>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-logging</artifactId>
    </exclusion>
</exclusions>

<!-- 引入log4j2 -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-log4j2</artifactId>
</dependency>
```
